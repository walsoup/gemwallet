## 2024-05-24 - Accessibility Labels for React Native Paper Icons
**Learning:** In this application, standard `react-native-paper` components that are purely icon-based (like `IconButton`, `FAB`, and `TextInput.Icon`) do not automatically inherit or generate context-aware accessibility descriptions. We must explicitly provide an `accessibilityLabel` prop, and critically, if the component acts as a toggle (like a play/stop button or hide/show password eye), the label *must* be dynamic to reflect the current state, otherwise screen readers convey stale or confusing information.
**Action:** When implementing new icon-only controls or toggles using `react-native-paper` components, always include a descriptive `accessibilityLabel` prop and ensure it reflects the component's active state if it's dynamic.
