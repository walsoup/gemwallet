## 2026-04-18 - Adding explicit accessibilityLabels to icon-only components in React Native Paper
**Learning:** Icon-only components in `react-native-paper` such as `IconButton`, `FAB`, and `TextInput.Icon` rely purely on their visual icon for meaning. For screen readers (like VoiceOver or TalkBack) to be able to describe these buttons to users with visual impairments, they require an explicit `accessibilityLabel` prop to be set. Without it, the screen reader may just read the component type or nothing at all, which makes the app confusing or unusable.
**Action:** Always ensure that when implementing icon-only buttons or inputs (e.g. `IconButton`, `FAB`, `TextInput.Icon`), a descriptive `accessibilityLabel` prop is provided. Also, dynamically update this label if the component's function changes (e.g., toggling a password visibility icon).
