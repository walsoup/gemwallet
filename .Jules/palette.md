## 2024-05-24 - Accessibility labels for dynamic IconButtons\n**Learning:** When adding accessibility labels to icon-only buttons that toggle state (like a record/stop button), the label must dynamically reflect the *action that will occur* when pressed, not just the icon itself. This provides crucial context for screen reader users.\n**Action:** Always verify if an icon button toggles state and update the `accessibilityLabel` dynamically to match the current intended action.
